# MediScan AI ⚕️

MediScan AI is a production-ready, clinical-grade AI-powered Medical Diagnosis Assistant web application. It empowers clinicians to upload chest radiograph scans (X-rays), receive real-time multi-disease classification (Normal, Pneumonia, Tuberculosis, COVID-19, Bronchitis, Lung Cancer), visualize explainable neural network regions via Grad-CAM heatmaps, consult an integrated generative AI assistant (Gemini), and export automated clinical PDF reports.

---

## 🛠️ Tech Stack Architecture

### Frontend
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS & Glassmorphism design tokens
- **Micro-Animations**: Framer Motion
- **Icons**: Lucide Icons
- **Visual Trends**: Custom responsive SVG vector charts & timelines

### Backend
- **Framework**: FastAPI (Python 3.13 compatible)
- **Server Gateway**: Uvicorn
- **AI Inference Engine**: TensorFlow / Keras (CNN) + OpenCV Computer Vision fallback
- **Report Compiler**: ReportLab PDF builder
- **Database Drivers**: PyMongo / Motor (Async driver)

### Database
- **Production**: MongoDB Atlas
- **Local/Development**: SQLite (automatic seamless fallback)

## 🚀 Step-by-Step Setup Instructions

### Prerequisites
- Node.js (v18+)
- Python (3.10 to 3.13)
- Pip (Python Package Manager)

---

### Step 1: Run the AI CNN Training Script
Before starting the backend, generate the `.h5` model file:
1. Open a terminal and navigate to the training directory:
   ```bash
   cd model_training
   ```
2. Install TensorFlow (if not already installed):
   ```bash
   pip install tensorflow numpy
   ```
3. Execute the training script to build the CNN and save it:
   ```bash
   python train_model.py
   ```
   *Note: This script generates a synthetic chest radiograph dataset, compiles the Sequential model, and writes `multi_disease_model.h5` to `backend/models/`.*

---

### Step 2: Set Up and Run the Backend API
1. Navigate to the backend directory:
   ```bash
   cd ../backend
   ```
2. Create and activate a Python virtual environment:
   ```bash
   python -m venv .venv
   # Windows Activation:
   .venv\Scripts\activate
   # macOS/Linux Activation:
   source .venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Configure environment variables (Optional `.env` file inside `backend/`):
   ```ini
   MONGODB_URL=mongodb+srv://<user>:<password>@cluster.mongodb.net/
   GEMINI_API_KEY=AIzaSy...
   JWT_SECRET=custom-long-security-token
   ```
   *Note: If no MONGODB_URL is configured, the system automatically spawns a local `mediscan.db` SQLite file. If no GEMINI_API_KEY is configured, the chatbot falls back to the smart clinical expert system.*
5. Run the FastAPI development server:
   ```bash
   uvicorn main:app --reload --port 8000
   ```
   *Verification: Open `http://localhost:8000/` in your browser. You should receive a JSON payload indicating status: "online".*

---

### Step 3: Run the Next.js Frontend
1. Navigate to the frontend directory:
   ```bash
   cd ../frontend
   ```
2. Install Node dependencies:
   ```bash
   npm install
   ```
3. Start the Next.js development server:
   ```bash
   npm run dev
   ```
4. Open your browser to `http://localhost:3000`.

## ☁️ Deployment Guide

### Database (MongoDB Atlas)
1. Register for a free tier database cluster on [MongoDB Atlas](https://www.mongodb.com/cloud/atlas).
2. Create a database user and allow access from all IP addresses (`0.0.0.0/0`) under Network Access.
3. Retrieve your application connection string: `mongodb+srv://<username>:<password>@cluster0.xxx.mongodb.net/?retryWrites=true&w=majority`.
4. Inject this connection string into your hosting service's environment variables (`MONGODB_URL`).

### Backend (Render or Railway)
1. Link your GitHub repository to [Render](https://render.com) or [Railway](https://railway.app).
2. Choose **Web Service** template and select the Python runtime environment.
3. Configure the Root Directory to `backend`.
4. Set the Build Command:
   ```bash
   pip install -r requirements.txt
   ```
5. Set the Start Command:
   ```bash
   uvicorn main:app --host 0.0.0.0 --port $PORT
   ```
6. Add your Environment variables in the dashboard (`MONGODB_URL`, `GEMINI_API_KEY`, `JWT_SECRET`).

### Frontend (Vercel)
1. Log into your [Vercel](https://vercel.com) workspace.
2. Select **Add New Project** and choose your repository.
3. Set the Root Directory to `frontend`.
4. Vercel automatically configures the Next.js build commands.
5. Hit **Deploy**.
